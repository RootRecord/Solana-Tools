Solana CLI + Python Capability Reference, 2026
==============================================

Purpose
-------
This is a broad 2026 reference for what can be done from Solana command-line tools
and Python code. It is no longer limited to the functions currently used in this
workspace.

It covers:
  - Core Solana CLI commands.
  - Solana Python RPC methods through solana-py.
  - Transaction, key, account, and instruction building through solders.
  - Classic SPL Token operations.
  - Token-2022 / Token Extensions operations, including mint and account extensions.
  - Practical Python patterns for building CLI scripts.

Version context used while writing:
  - Solana / Agave CLI docs show solana-cli 3.1.x era commands.
  - solana-py docs show solana 0.36.x era APIs.
  - solders docs show solders 0.27.x era APIs.
  - Token-2022 docs describe the Token Extensions Program and spl-token support.

Always confirm exact flags on your machine with:
  solana --help
  solana <command> --help
  solana-keygen --help
  spl-token --help
  spl-token <command> --help

Key Concepts
------------

SOL and lamports:
  1 SOL = 1,000,000,000 lamports.

Commitment levels:
  processed  - newest local node view, can roll back.
  confirmed  - voted on by supermajority stake, more stable.
  finalized  - strongest normal confirmation state.

Clusters:
  mainnet-beta - production.
  devnet       - developer testing with faucet SOL.
  testnet      - validator testing.
  localhost    - local validator or Surfpool.

Accounts:
  Every Solana account has lamports, owner program, data, executable flag, and rent info.

Transactions:
  A transaction contains one or more instructions, a recent blockhash, a fee payer, and signatures.
  In 2026, prefer versioned transactions with MessageV0 unless you specifically need legacy format.

Address Lookup Tables:
  Address lookup tables let versioned transactions reference more accounts than fit directly in the
  transaction message.

Token Accounts:
  Wallets do not hold SPL tokens directly. A token balance is stored in a token account.

Associated Token Accounts:
  The ATA convention derives a canonical token account from:
    owner wallet + token program ID + mint
  Classic Token and Token-2022 ATAs differ because the token program ID is part of the seed list.


Python Packages
---------------

solana:
  Network/RPC client package. Main imports:
    from solana.rpc.api import Client
    from solana.rpc.async_api import AsyncClient
    from solana.rpc.types import TxOpts, TokenAccountOpts, MemcmpOpts, DataSliceOpts

solders:
  Fast Python bindings for Solana SDK types. Main imports:
    from solders.keypair import Keypair
    from solders.pubkey import Pubkey
    from solders.hash import Hash
    from solders.signature import Signature
    from solders.instruction import Instruction, AccountMeta
    from solders.message import Message, MessageV0, to_bytes_versioned
    from solders.transaction import Transaction, VersionedTransaction
    from solders.system_program import transfer, create_account, allocate, assign
    from solders.compute_budget import set_compute_unit_limit, set_compute_unit_price

spl.token:
  Python helpers for classic SPL Token instructions. Main imports:
    from spl.token.constants import TOKEN_PROGRAM_ID, ASSOCIATED_TOKEN_PROGRAM_ID
    from spl.token.instructions import create_associated_token_account
    from spl.token.instructions import mint_to, burn, transfer_checked
    from spl.token.instructions import approve, revoke, close_account
    from spl.token.instructions import freeze_account, thaw_account
    from spl.token.instructions import set_authority

Token-2022 in Python:
  Many classic SPL Token helper layouts are byte-compatible with Token-2022 when you pass the
  Token-2022 program ID. New Token-2022 extension instructions may require newer package support,
  raw Instruction building, the spl-token CLI, TypeScript, or Rust depending on your installed
  Python SPL library.


Solana CLI Commands
-------------------

Global command shape:
  solana [FLAGS] [OPTIONS] <SUBCOMMAND>

Common global options:
  --url / -u <URL_OR_MONIKER>       Select mainnet-beta, devnet, testnet, localhost, or custom RPC.
  --keypair / -k <KEYPAIR>          Signer keypair path, ASK, stdin keypair, or hardware wallet URL.
  --commitment <LEVEL>              processed, confirmed, finalized.
  --output <FORMAT>                 json or json-compact.
  --skip-preflight                  Send without RPC preflight simulation.
  --use-quic / --use-tpu-client     Transaction sending transport options.

Account and wallet commands:
  solana address
    Print the configured wallet public key.

  solana account <ADDRESS>
    Show account owner, lamports, executable flag, data length, and optional raw data output.

  solana balance [ADDRESS]
    Show SOL balance for configured wallet or another account.

    Transfer SOL between system accounts.

  solana airdrop <AMOUNT> [RECIPIENT]

  bytes(keypair)

  transfer_checked(TransferCheckedParams(program_id, source, mint, dest, owner, amount, decimals))

Delegates:
  approve(ApproveParams(...))
  approve_checked(...)
  revoke(RevokeParams(...))

Freeze/thaw:
  freeze_account(FreezeAccountParams(program_id, account, mint, authority))
  thaw_account(ThawAccountParams(program_id, account, mint, authority))

Close:
  close_account(CloseAccountParams(program_id, account, dest, owner))

Authorities:
  set_authority(SetAuthorityParams(program_id, account, authority, current_authority, new_authority))

Authority types:
  AuthorityType.MINT_TOKENS
  AuthorityType.FREEZE_ACCOUNT
  AuthorityType.ACCOUNT_OWNER
  AuthorityType.CLOSE_ACCOUNT

Sync native SOL:
  sync_native(SyncNativeParams(...))


Token-2022 / Token Extensions Overview
--------------------------------------

Token-2022 is a superset of the classic Token Program:
  - Classic token instructions are byte-compatible.
  - New features are opt-in extensions.
  - Extension state is stored after the base mint/account data.
  - Many extensions must be enabled when creating the mint or account.
  - Token-2022 uses a different program ID, so ATAs are different from classic token ATAs.

Use Token-2022 with spl-token:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb <command>

Python constant:
  TOKEN_2022_PROGRAM_ID = Pubkey.from_string("TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb")

Token-2022 ATA derivation:
  ata = Pubkey.find_program_address(
      [bytes(owner), bytes(TOKEN_2022_PROGRAM_ID), bytes(mint)],
      ASSOCIATED_TOKEN_PROGRAM_ID,
  )[0]


Token-2022 Core Operations
--------------------------

Create Token-2022 mint:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token

Create Token-2022 token account:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-account <MINT>

Mint Token-2022 tokens:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb mint <MINT> <AMOUNT> [RECIPIENT]

Transfer Token-2022 tokens:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb transfer <MINT> <AMOUNT> <RECIPIENT>

Burn Token-2022 tokens:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb burn <ACCOUNT_OR_MINT> <AMOUNT>

Freeze/thaw Token-2022 accounts:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb freeze <ACCOUNT>
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb thaw <ACCOUNT>

Close Token-2022 token account:
  spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb close <ACCOUNT>


Token-2022 Extensions
---------------------

Mint Close Authority:
  Purpose:
    Allows a mint account to be closed and rent reclaimed when supply is zero.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-close
    spl-token close-mint <MINT>
  Notes:
    Closing/reinitializing mints with different extensions can create integration hazards.

Transfer Fees:
  Purpose:
    Protocol-level transfer fee withheld from recipient account.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token \
      --transfer-fee-basis-points <BPS> \
      --transfer-fee-maximum-fee <MAX_FEE>
    spl-token transfer <MINT> <AMOUNT> <RECIPIENT> --expected-fee <FEE>
    spl-token withdraw-withheld-tokens <DESTINATION_ACCOUNT> <SOURCE_ACCOUNT>...
    spl-token withdraw-withheld-tokens --include-mint <DESTINATION_ACCOUNT>
  Notes:
    Use transfer_checked or transfer_checked_with_fee. Plain unchecked transfer can fail.
    Withheld fees can sit in token accounts or be harvested to the mint.

Default Account State:
  Purpose:
    Makes newly created accounts initialized or frozen by default.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token \
      --enable-freeze --default-account-state frozen
    spl-token update-default-account-state <MINT> initialized
    spl-token update-default-account-state <MINT> frozen
  Notes:
    Useful for permissioned tokens where accounts must be thawed before use.

Immutable Owner:
  Purpose:
    Prevents token account owner reassignment.
  CLI:
    spl-token create-account <MINT> --immutable
  Notes:
    Associated token accounts for Token-2022 are always immutable.

Non-Transferable Tokens:
  Purpose:
    Soulbound/non-transferable tokens. Holders can still burn/close where allowed.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-non-transferable

Required Memo on Transfer:
  Purpose:
    Incoming transfers to the account must include a memo instruction immediately before transfer.
  CLI:
    spl-token enable-required-transfer-memos <TOKEN_ACCOUNT>
    spl-token disable-required-transfer-memos <TOKEN_ACCOUNT>
    spl-token transfer <MINT> <AMOUNT> <RECIPIENT> --with-memo "memo text"

Reallocate:
  Purpose:
    Expands token account data to fit extensions added after account creation.
  CLI:
    spl-token enable-required-transfer-memos <TOKEN_ACCOUNT>
  Notes:
    The CLI may automatically add reallocation when enabling compatible account extensions.

Interest-Bearing Mint:
  Purpose:
    Display/UI amount changes over time according to interest rate. Raw token amount does not change.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --interest-rate <BPS>
    spl-token set-interest-rate <MINT> <BPS>
  Notes:
    This is cosmetic accounting, not automatic minting.

Permanent Delegate:
  Purpose:
    Mint-level delegate that can transfer or burn from any account for that mint.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-permanent-delegate
    spl-token authorize <MINT> permanent-delegate <NEW_DELEGATE>
  Notes:
    Useful for regulated assets; dangerous if misused.

CPI Guard:
  Purpose:
    Restricts token actions performed through cross-program invocation.
  CLI:
    spl-token enable-cpi-guard <TOKEN_ACCOUNT>
    spl-token disable-cpi-guard <TOKEN_ACCOUNT>
  Effects during CPI:
    Transfer requires delegate authority.
    Burn requires delegate authority.
    Approve is prohibited.
    Close destination must be account owner.
    Set close authority is prohibited unless unsetting.
    Set owner is always prohibited.

Transfer Hook:
  Purpose:
    Calls a configured external hook program on transfers.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --transfer-hook <PROGRAM_ID>
    spl-token set-transfer-hook <MINT> <PROGRAM_ID>
  Extra metas helper:
    spl-transfer-hook create-extra-metas <PROGRAM_ID> <MINT_ID> [<ACCOUNT_PUBKEY>:<ROLE> ...]
  Notes:
    Requires a deployed program implementing the transfer-hook interface.

Metadata Pointer:
  Purpose:
    Points a mint to canonical metadata account.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --metadata-address <ADDRESS>
  Notes:
    Clients should verify that mint and metadata point to each other.

Token Metadata Extension:
  Purpose:
    Stores token metadata directly in the Token-2022 mint account.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-metadata
    spl-token initialize-metadata <MINT> <NAME> <SYMBOL> <URI> --update-authority <AUTHORITY>
    spl-token update-metadata <MINT> name <NEW_NAME>
    spl-token update-metadata <MINT> symbol <NEW_SYMBOL>
    spl-token update-metadata <MINT> uri <NEW_URI>
    spl-token update-metadata <MINT> <CUSTOM_FIELD> <VALUE>
    spl-token update-metadata <MINT> <CUSTOM_FIELD> --remove

Group Pointer:
  Purpose:
    Points a mint to canonical group configuration.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --group-address <ADDRESS>

Group:
  Purpose:
    Stores group configuration directly in mint. Useful for collection-like relationships.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-group
    spl-token initialize-group <MINT> <MAX_SIZE> --update-authority <AUTHORITY>

Member Pointer:
  Purpose:
    Points a member mint to canonical member configuration.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --member-address <ADDRESS>

Member:
  Purpose:
    Stores group membership configuration directly in mint.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-member
    spl-token initialize-member <MINT> --update-authority <AUTHORITY>

Scaled UI Amount:
  Purpose:
    Applies a UI multiplier to raw amounts. Raw token amount does not change.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --ui-amount-multiplier <MULTIPLIER>
    spl-token update-ui-amount-multiplier <MINT> <MULTIPLIER> [UNIX_TIMESTAMP]

Pausable:
  Purpose:
    Pause mint, burn, and transfer activity.
  CLI:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-pause
    spl-token pause <MINT>
    spl-token resume <MINT>

Confidential Transfers:
  Purpose:
    Hide transfer amounts with encrypted balances and zero-knowledge proof flows.
  CLI examples vary by installed spl-token version:
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-confidential-transfers auto
    spl-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb create-token --enable-confidential-transfers manual
  Notes:
    Must be enabled at mint creation.
    Standard transfer is not the confidential transfer flow.
    Some confidential operations require proof generation tooling.

Confidential Transfer Fee:
  Purpose:
    Transfer fees for confidential transfers.
  Notes:
    Must be initialized with confidential transfer support.
    Tooling support depends on installed spl-token and proof-generation packages.

Confidential Mint/Burn:
  Purpose:
    Confidentially mint or burn amounts where supported by the Token-2022 program/tooling.
  Notes:
    Requires compatible client tooling and proof generation.


Token-2022 Python Patterns
--------------------------

Create Token-2022 ATA:
  TOKEN_2022_PROGRAM_ID = Pubkey.from_string("TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb")
  ASSOCIATED_TOKEN_PROGRAM_ID = Pubkey.from_string("ATokenGPvbdGVxr1b2hvZbsiqW5xWH25efTNsLJA8knL")
  ata = Pubkey.find_program_address(
      [bytes(owner), bytes(TOKEN_2022_PROGRAM_ID), bytes(mint)],
      ASSOCIATED_TOKEN_PROGRAM_ID,
  )[0]

Read Token-2022 token balance:
  client.get_token_account_balance(ata)

Find Token-2022 accounts by owner:
  from solana.rpc.types import TokenAccountOpts
  opts = TokenAccountOpts(program_id=TOKEN_2022_PROGRAM_ID)
  client.get_token_accounts_by_owner(owner, opts)

Check whether an account is Token-2022:
  info = client.get_account_info(token_account).value
  is_token_2022 = info.owner == TOKEN_2022_PROGRAM_ID

Build classic-layout Token-2022 mint_to:
  ix = mint_to(MintToParams(
      program_id=TOKEN_2022_PROGRAM_ID,
      mint=mint,
      dest=destination_token_account,
      mint_authority=authority.pubkey(),
      amount=raw_amount,
  ))

Build classic-layout Token-2022 burn:
  ix = burn(BurnParams(
      program_id=TOKEN_2022_PROGRAM_ID,
      account=token_account,
      mint=mint,
      owner=owner.pubkey(),
      amount=raw_amount,
  ))

Build classic-layout Token-2022 set_authority:
  ix = set_authority(SetAuthorityParams(
      program_id=TOKEN_2022_PROGRAM_ID,
      account=mint,
      authority=AuthorityType.MINT_TOKENS,
      current_authority=current_authority.pubkey(),
      new_authority=new_authority_pubkey,
  ))

Build classic-layout Token-2022 thaw:
  ix = thaw_account(ThawAccountParams(
      program_id=TOKEN_2022_PROGRAM_ID,
      account=token_account,
      mint=mint,
      authority=freeze_authority.pubkey(),
  ))

Send any built instruction:
  blockhash = client.get_latest_blockhash().value.blockhash
  msg = MessageV0.try_compile(
      payer=payer.pubkey(),
      instructions=[ix],
      address_lookup_table_accounts=[],
      recent_blockhash=blockhash,
  )
  tx = VersionedTransaction(msg, [payer])
  sig = client.send_raw_transaction(bytes(tx), opts=TxOpts(skip_preflight=False))


Metaplex Metadata vs Token-2022 Metadata
----------------------------------------

Metaplex Token Metadata:
  - Separate metadata PDA account.
  - Works with classic SPL Token and Token-2022.
  - Widely supported by wallets, explorers, and marketplaces.
  - Program ID: metaqbxxUerdq28cj1RbAWkYQm3ybzjb6a8bt518x1s

Token-2022 Metadata Extension:
  - Metadata stored directly in mint account.
  - Only for Token-2022 mints created with metadata extension.
  - Uses metadata pointer plus token metadata extension.

Python can build Metaplex instructions manually with:
  Instruction(program_id=METADATA_PROGRAM_ID, accounts=[...], data=...)

For production metadata, prefer a maintained Metaplex Python client or build instructions from
the current Metaplex program spec rather than guessing instruction bytes.


Common Python CLI Script Patterns
---------------------------------

Load keypair:
  import base58
  payer = Keypair.from_bytes(base58.b58decode(PRIVATE_KEY_BASE58))

Create RPC client:
  client = Client(RPC_URL)

Get SOL balance:
  sol = client.get_balance(payer.pubkey()).value / 1_000_000_000

Get token supply:
  supply = client.get_token_supply(mint).value

Get token account balance:
  balance = client.get_token_account_balance(token_account).value

Get latest blockhash:
  blockhash = client.get_latest_blockhash().value.blockhash

Compile/send transaction:
  msg = MessageV0.try_compile(payer.pubkey(), instructions, [], blockhash)
  tx = VersionedTransaction(msg, [payer])
  result = client.send_raw_transaction(bytes(tx), opts=TxOpts(skip_preflight=False))

Add memo:
  Build a Memo program instruction before the transfer instruction.

Add priority fee:
  instructions = [
      set_compute_unit_price(micro_lamports),
      set_compute_unit_limit(units),
      real_instruction,
  ]

Use retries:
  Retry getting blockhash/quote/sending, but do not blindly re-send successful transactions.

Use simulation:
  client.simulate_transaction(tx)

Confirm:
  client.confirm_transaction(signature, commitment="confirmed")


Production Safety Checklist
---------------------------

Never hard-code private keys in scripts.
  Use environment variables, encrypted secret stores, hardware wallets, or keypair files.

Use devnet first.
  Test mint, transfer, burn, authority, and extension flows before mainnet.

Use checked token instructions.
  Prefer transfer_checked, mint_to_checked, burn_checked where available.

For Token-2022 transfer fees:
  Use transfer_checked_with_fee or CLI --expected-fee where available.

For Token-2022 metadata/group/member pointers:
  Verify both sides of pointer relationships when reading.

For authority changes:
  Understand irreversible --disable operations.

For freeze authority:
  Disabling freeze authority means frozen accounts may remain frozen forever.

For close operations:
  Token accounts usually must be empty.
  Mint close requires Token-2022 MintCloseAuthority and zero supply.

For program deploys:
  Keep upgrade authority secure.
  Consider final authority revocation only after audits and confidence.

For RPC:
  Public endpoints are rate-limited and not ideal for production.
  Use a dedicated RPC provider for bots, trading, indexers, and production apps.
